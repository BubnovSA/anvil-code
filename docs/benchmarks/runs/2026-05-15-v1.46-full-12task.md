## Regression analysis

**T2 getHTTPStatusCode:** was ✅ in v1.43. Now `test_fail` — Coder implementation correct but TesterAgent tests fail. Or: 3-hop expansion adds noise to context, Coder picks wrong file to edit.

**T3 onError callback:** Reviewer rejects with "onError not added to interfaces" — Coder implemented partial feature (only routes handler, not type definitions). The 3-hop BFS might be flooding context with unrelated caller symbols from nodeHTTPRequestHandler callsites, distracting Coder from the type interface.

**T5 maxBodySize:** `test_fail` — same pattern as T2, was passing before.

**Root cause hypothesis:** `RAG_GRAPH_HOPS=3` adds significant caller context (3 levels deep in trpc monorepo = many symbols from unrelated packages). For trpc with 3938 symbols and complex inter-package dependencies, 3-hop BFS surfaces too much cross-package noise. Token budget may be exhausted with irrelevant callers before the actually relevant context appears.

**Fix:** Reduce `RAG_GRAPH_HOPS` for production. Default should be 2 or even 1 for large monorepos, or add a per-file limit (max 3 callers per symbol at each hop level).
