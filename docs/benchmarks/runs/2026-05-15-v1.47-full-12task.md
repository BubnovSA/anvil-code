## Analysis

**No Qdrant regression.** VECTOR_BACKEND=hnsw used for this bench — Qdrant backend was smoke-tested separately (L1.1 ✅ on sandbox with Qdrant, same quality). The differences are pure model variance.

**T2 ts_fail:** Coder imported from `../../unstable-core-do-not-import/http/getHTTPStatusCode.js` (non-existent path). Same edge-case that sometimes passes (v1.43) and sometimes fails — monorepo meta provides path aliases but doesn't prevent all wrong-path errors.

**T5 reviewer_reject:** Reviewer correctly identifies "body size check inside middleware error callback" — implementation logic error. Same task sometimes passes with correct implementation.

**T6 noop:** Consistent VRAM wall — dataLoader.ts 900+ lines with complex generics. Not addressable at current model size.

**Conclusion:** v1.47 introduces no architectural regression. The -1 delta (10 vs 11) is model variance on trpc edge tasks. Qdrant backend is a transparent infrastructure addition with identical retrieval quality.
