## What v1.24 changed (under test)

- `applyEdits` now tries strict match first (unchanged), then a whitespace-tolerant fallback: search is split on `\s+`, segments escaped, joined with `\s+` regex. Replace remains literal. Match must still be unique under the tolerant pattern (zero or ≥2 matches → abort).
- `ApplyResult` shape: ok-branch gains `tolerantEdits: number[]` listing which edit indices required the fallback. Strict-only paths return `[]`.
- `SafeWriter` emits `logger.warn({ tolerantEditIndices }, '… whitespace-tolerant fallback …')` whenever any edit fell back, so run files surface "model minified search blocks" without ad-hoc log mining.
- 9 new unit tests in `edit-applier.test.ts` cover: minified one-line→multi-line, tabs↔spaces, regex special chars escaped, ambiguity rejection under tolerant pattern, strict-wins-when-both-possible, replace stays verbatim, whitespace-only search guarded out, mixed strict+tolerant edits.
- 205/205 unit tests green; 12/12 packages build.

## What worked

- **applyEdits / SafeWriter v1.24 changes are correct and well-covered.** 9 new unit tests (20 total in `edit-applier.test.ts`); 205/205 across the suite; 12/12 packages build. Strict path is byte-identical to v1.23, no behavioral regression on existing kitchens.
- **Conservative tolerant fallback held the line.** It did not produce a single false-positive match across three real-world model runs. Whitespace-only search guard works.
- L1.1 still lands cleanly with `qwen2.5-coder:32b`.

## What broke

- **L2.1 regressed against the 2026-04-27 baseline** (10/10 → 5.2/10) purely from model variance. Same model, same prompt, same code (modulo non-firing tolerant). The model forgot to add the Fastify module-augmentation for `request.start`, and three Fixer rounds didn't recover it. This is not v1.24's fault, but it is a reproducibility data point — single-shot benchmarks understate the noise floor.
- **L2.3 cumulative landed as a partial commit, with the validator passing because typecheck saw the uncommitted-but-on-disk state.** The commit's `writtenFiles` list and the validation snapshot diverged. The user has no way to know from the `done` event that one of three files didn't make it into the commit.
- **Tolerant fallback didn't get a chance to fire.** None of the three runs hit the targeted "search minified to one line" failure mode. Both L1.1 retry and L2.3 retries were on structurally-different (paraphrased / hallucinated) search blocks, where `\s+` normalization correctly couldn't help.

## Lessons / next iteration ideas

1. **v1.24 is implemented correctly but the failure mode it targets is not the dominant one for `qwen2.5-coder:32b` in this snapshot.** It's an insurance policy — cheap to keep, it'll catch the minification mode if it shows up later. No code to revert.
2. **Silent partial completion is now reproduced live** (L2.3 here, in addition to L2.2 in the 2026-04-27 audit). v1.28 should be promoted in importance: the run scored 6.7/10 partly *because* the user has no signal that the task half-landed. Adding `commit_partial` and `incomplete_files: string[]` to the `done` payload would directly reflect this gap to the operator.
3. **Validation/commit divergence** is a deeper bug than v1.28 alone fixes. The validation Fixer rounds write to disk and update `allFileChanges`, but the *commit's* file list comes from `writtenFiles[]` which is populated only during the initial+retry-with-real-content phase. Files written exclusively during validation Fixer rounds don't make the commit. Worth documenting as a v1.28a follow-up: extend `writtenFiles` from validation-loop disk writes too, OR (cleaner) have the commit step just `git add -A` the working tree subject to `change.path`-allowlist.
4. **L2.1 regression reinforces "model size dominates prompts" but adds nuance: model variance also dominates a single-shot benchmark.** A future iteration should run each task ≥2 times and report mean+spread, otherwise we're chasing noise. Cheap on local Ollama.
5. **Tolerant fallback test coverage is solid in unit tests, but no in-vivo activation yet.** Next benchmark cycle, before declaring tolerant matters, deliberately seed a known-minified search via a Coder prompt experiment to confirm the path activates end-to-end. Until then, count it as a structural safety net, not a measured win.

## Decision

Keep v1.24 in tree. Move on to v1.25 (repo-map). Re-run this same suite after v1.25 and v1.28 — partial-completion visibility plus better contextual grounding are likely to produce more meaningful score deltas than tolerant matching alone.
